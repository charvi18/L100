# Git Advanced Notes

## What is Git?
Git is a distributed version control system used to track changes in code and collaborate efficiently.

## Git Architecture
- Working Directory
- Staging Area
- Repository

## Three States of Git

1. Working Directory → Files you edit locally
2. Staging Area → Files prepared for next commit
3. Repository → Permanent history stored in Git

This explains how Git tracks changes logically.

 Example Workflow

1. Modify file in working directory
2. Stage changes
   git add architecture.txt
3. Commit snapshot
   git commit -m Update architecture notes

This workflow shows how data moves from working directory → staging → repository
